# Smart Monitor v2 設計規格

## 背景

現有 smart-monitor 已完成即時層（Fugle WebSocket + Discord 推播）。
v2 目標：升級為多使用者交易助理系統，透過 LINE Bot 私訊操作，加入波段分析層。

---

## 系統架構

```
使用者 LINE 私訊
        ↓
LINE Bot Webhook Server（FastAPI，本機 Mac 常駐）
        ↓ 僅設定監控時呼叫一次
Claude API（自然語言 → 結構化監控參數）
        ↓
監控管理器（MonitorManager）
  ├── 每位使用者獨立監控實例
  ├── 即時層（Fugle WebSocket，條件達成才通知）
  └── 波段層（08:30 盤前 / 13:35 盤後，排程觸發）
        ↓
LINE Bot Push Message → 使用者私訊
```

---

## 三個子專案（依序實作）

### P1：波段分析層

獨立可執行，不依賴 LINE Bot。

**新增檔案：**

```
daily_data.py       富果 REST 抓日 K（RestClient）
swing_strategy.py   均線 + 高點回撤判斷
analyze.py          入口，手動執行或排程觸發
```

**現有檔案完全不動：**
`main.py`, `market_data.py`, `strategy.py`, `notifier.py`

**波段分析邏輯：**

| 指標 | 參數 | 黃燈（預警） | 紅燈（警示） |
|------|------|------------|------------|
| 20 日均線 | MA20 | 距 MA20 < 2% | 收盤跌破 MA20 |
| 高點回撤 | 近 20 日最高收盤 | 回撤 5~8% | 回撤 ≥ 8% |

**config.json 新增欄位：**
```json
{
  "swing_ma_days": 20,
  "swing_pullback_pct": 8.0,
  "swing_pullback_warn_pct": 5.0,
  "swing_ma_warn_pct": 2.0,
  "swing_lookback_days": 20
}
```

**執行時機：**
- `08:30`：盤前分析，推播今日技術面摘要與訊號
- `13:35`：盤後分析，推播當日結果與明日建議
- 手動：`python analyze.py` 隨時可執行

**盤前分析輸出格式：**
```
📊 盤前分析｜2026-06-05 08:30
───────────────────────────
【3312 弘憶】
  昨收  68.50 元
  MA20  66.20 元（+3.47%，位於均線上方）
  高點  71.00 元（回撤 3.52%）
  訊號  ✅ 無異常，可續抱
───────────────────────────
```

**盤後分析輸出格式：**
```
📊 盤後分析｜2026-06-05 13:35
───────────────────────────
【3312 弘憶】
  今收  67.80 元（-1.02%）
  MA20  66.30 元（+2.26%）
  高點  71.00 元（回撤 4.51%）
  訊號  🟡 留意：距高點回撤接近警戒（5%）
  明日  續抱，若跌破 66.30 考慮減碼
───────────────────────────
```

### P2：LINE Bot + Claude 解析

**技術棧：**
- `FastAPI`：Webhook server
- `ngrok`：本機公開 URL
- `claude-3-haiku`：自然語言解析（最便宜，夠用）
- 使用者資料：`users/{line_user_id}/config.json`

**使用者加好友流程：**
```
加好友
  ↓
Bot 推播歡迎訊息：
「👋 歡迎！輸入你想監控的股票條件即可開始。
  例如：監控弘憶，5 張，均價 64.86，停損 63」
```

**對話狀態機：**

```
IDLE
  ↓ 收到含監控關鍵字的訊息
PARSING（呼叫 Claude API 一次）
  ↓ 解析完成，缺欄位
COLLECTING（純程式邏輯追問）
  ↓ 欄位齊全
CONFIRMING（顯示確認卡片）
  ↓ 使用者確認
MONITORING（監控執行中）
  ↓ /停止 或 停損/清倉觸發
IDLE
```

**Claude API 呼叫規則：**
- 觸發條件：訊息包含關鍵字（監控、持股、買進、追蹤）
- 每次設定只呼叫一次
- 其他所有互動不呼叫 Claude
- 每位使用者每日上限：10 次設定（防濫用）

**Claude 解析的 System Prompt 職責：**
從自然語言抽取以下欄位，回傳 JSON：
```json
{
  "stock_id": "3312",
  "stock_name": "弘憶",
  "total_shares": 5000,
  "cost_price": 64.86,
  "stop_loss_moving": 63.0,
  "target_stage_1": 75.0,
  "target_stage_2": null
}
```
無法確定的欄位回傳 `null`，不猜測。

**確認卡片格式：**
```
📋 請確認監控條件

股票：3312 弘憶
持股：5 張（25,000 股）
均價：64.86 元
停損：63.00 元（-2.86%）
目標一：75.00 元（+15.65%）
目標二：未設定

[✅ 確認開始監控]  [❌ 重新輸入]
```

**管理指令（關鍵字，不呼叫 Claude）：**

| 使用者輸入 | Bot 動作 |
|-----------|---------|
| `狀態` / `status` | 顯示目前監控條件與今日損益 |
| `修改停損 62` | 確認後即時更新，5 秒內生效 |
| `修改目標 80` | 確認後即時更新 |
| `停止` / `stop` | 停止監控，清除排程 |
| 其他訊息 | 固定回覆提示可用指令，不呼叫 Claude |

Bot 每次動作完都附上：
```
可用指令：狀態 ／ 修改停損 X ／ 修改目標 X ／ 停止
```

## 監控生命週期

```
設定啟動
  ↓
即時層常駐（WebSocket）
  ↓ 每天自動
  08:30 盤前分析推播
  13:35 盤後分析推播
  ↓ 任何時間
  條件達成 → 立刻推播
  ↓ 結束條件（任一）
  /停止（手動）
  停損觸發（自動停止）
  目標二清倉觸發（自動停止）
```

**超過 08:30 才設定：**
不補推盤前分析，改為立刻推播「即時現況快照」：
```
⚡ 即時現況（10:00 設定）
  現價 67.2 元，今日 +2.3%
  MA20 65.8 元（+2.1%）
  回撤 1.5%
  ⚠️ 今日盤前分析已過，明日 08:30 恢復
```

**停止後隔天：**
需重新輸入設定，不自動恢復（避免使用者忘記已停止）。
