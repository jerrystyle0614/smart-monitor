# 選股推薦功能設計規格

## 核心設計理念

投資人輸入需求（資金、期間、風險偏好）→ AI 判斷策略 → 技術面篩選 → 推薦 5 檔

**Why:** 讓不懂技術面的一般投資人只需回答 3 個問題，就能得到適合自己的選股建議。

**How to apply:** 所有選股邏輯圍繞這三個維度展開，不要加入複雜的使用者操作。

---

## 問答流程（3 題）

```
Q1：可投入資金（元）→ 輸入數字
Q2：持有期間 → 1 短期(1~4週) / 2 中期(1~3月) / 3 長期/定額
Q3：風險承受度 → 1 保守(虧5%) / 2 穩健(虧10~15%) / 3 積極(大波動)
```

---

## 策略對應邏輯

| 組合 | 策略類型 | 篩選重點 |
|------|---------|---------|
| 保守 + 長期 | high_yield_stable | ETF 優先、低波動 |
| 保守 + 短/中期 | defensive_band | MA20 上方、低回撤、大型股 |
| 穩健 + 短/中期 | momentum_mid | MA20 向上、突破訊號 |
| 積極 + 任何 | aggressive_short | 強勢突破、放量 |
| 任何 + 長期/定額 | dca_stable | ETF 或績優大型股 |

---

## 技術面篩選參數

| 策略 | MA需在均線上 | 最大回撤 | 最低量比 |
|------|------------|---------|---------|
| aggressive_short | ✅ | 12% | 1.1x |
| momentum_mid | ✅ | 8% | 1.0x |
| defensive_band | ✅ | 5% | 0.8x |
| high_yield_stable | ❌ | 8% | 0.5x |
| dca_stable | ❌ | 10% | 0.5x |

## 資金篩選邏輯（Q4 決策：B 方案）

- 整張門檻 = 股價 × 1000
- 資金 >= 整張門檻 → 顯示「✅ 可買整張（約 XX 元）」
- 資金 < 整張門檻 → 顯示「⚠️ 零股｜整張需約 XX 萬元」
- **不過濾掉零股股票，讓使用者自己決定能不能買**

**Why:** 使用者明確要求推薦零股，但讓使用者自行判斷是否買入。

## 快取機制

- Cache key = `{date}_{risk}_{period}_{capital_tier}`（MD5 前 12 碼）
- Capital tier 分 4 級：under1w / 1w_5w / 5w_20w / over20w
- 同一天同一策略組合 → 回傳相同結果（節省 API 費用）
- 每天自動失效（key 含日期）

---

## 每日查詢上限

- 每個 uid 每天最多 **10 次**
- 超過回覆：「今日查詢次數已達上限（10/10），明日再試」
- 查詢記錄存在 `users/{uid}/picker_queries.json`，含日期欄位跨天自動重置


## 掃描股票池

固定清單（`_SCAN_UNIVERSE`）涵蓋：
- 大型權值股：台積電、鴻海、聯發科、廣達等
- ETF：0050、0056、00878、00940、00919、006208
- 中型科技股：瑞昱、聯詠、大立光、華碩等
- 金融股：富邦金、國泰金、兆豐金、合庫金等
- 低價股：中鋼、台塑、南亞等

**Why:** 避免掃描全市場（速度太慢），用固定池覆蓋各類型股票。

**How to apply:** 未來可在此擴充 FinMind 法人資料篩選（Phase B 進階）。
